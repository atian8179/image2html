# Visual Audit Checklist

Use this checklist before coding and after screenshot verification.

## Composition

- Identify the exact viewport or canvas ratio.
- Mark the largest regions first.
- Note where the primary action is.
- Note whether the design is centered, split, sidebar-based, card-based, full-bleed, or layered.

## Layout

- Estimate columns, rows, gutters, and margins.
- Record repeated card or list item dimensions.
- Identify fixed-aspect areas such as phones, videos, dashboards, charts, and canvases.
- Check whether panels align to a shared grid.

## Typography

- Capture visible text exactly when legible.
- Estimate heading/body/caption sizes.
- Note font weight, casing, line height, and alignment.
- Preserve hierarchy even when exact font is unknown.

## Color And Material

- Sample or estimate background, surface, text, border, accent, warning, and success colors.
- Note gradients, blur, glass, shadows, strokes, highlights, and opacity.
- Avoid shifting the palette to a generic theme.

## Components

- List all buttons, tabs, chips, cards, fields, sliders, toggles, icons, nav items, and menus.
- Mark selected, disabled, hover, pressed, and focus states if visible or implied.
- Identify repeated components that should share CSS classes.

## Responsive Risks

- Longest visible text.
- Dense toolbars.
- Fixed-position panels.
- Large images or canvases.
- Cards that may collapse poorly.
- Buttons whose labels may clip on mobile.

## Final Screenshot Review

- Does the page read as the same product at a glance?
- Are the main regions the same size and position?
- Are colors and contrast close?
- Is text legible and unclipped?
- Are assets present and cropped correctly?
- Is there horizontal overflow?
- Do interactive controls visibly respond?
